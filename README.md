**ovpn-cert-tg-gen — что это**
Сервис/бот на Python, который генерирует **клиентские сертификаты и `.ovpn` конфигурации** для OpenVPN и **отправляет их в Telegram** через бота. Работает как системный сервис. ([GitHub][1])

# Что требуется перед использованием

1. OpenVPN с действующей PKI (CA, server).
2. Telegram-бот и его токен (`BOT_TOKEN`).
3. Telegram-чат/группа и их `CHAT_ID`.
4. Настройки в `.env` по примеру из `.env.example` (токен, чат и др.). ([GitHub][2])

# Как настроить

1. Скопировать `.env.example → .env` и вставить:
   • `BOT_TOKEN` — токен Telegram-бота
   • `CHAT_ID` — ID чата/группы куда будут отправляться конфиги ([GitHub][2])

2. Установить зависимости Python из `requirements.txt`. ([GitHub][1])

3. Развернуть как системный сервис:
   • файл `ecos-tg-bot-ovpn.service` — unit для запуска бота в фоне через systemd (авто-рестарт, логирование). ([GitHub][3])

# Что делает сервис
Когда запускается, он:
• слушает команды/запросы через Telegram; ([GitHub][1])
• при запросе генерирует клиентские ключи/сертификат; ([GitHub][1])
• собирает `.ovpn` файл с включенными cert/key; ([GitHub][1])
• отправляет `.ovpn` файл в Telegram пользователю/группе. ([GitHub][1])

# Запуск

1. Запускаешь сервис (`systemctl start ecos-tg-bot-ovpn.service`). ([GitHub][3])
2. В Telegram через бот даёшь команду (например, имя нового клиента). ([GitHub][1])
3. В чат приходит готовый `.ovpn` для скачивания и использования. ([GitHub][1])
